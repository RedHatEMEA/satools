About SA tools presentation search

This is the help page for SA tools presentation search 0.4, written by Jim Minter. SA tools presentation search and all its command line tooling are open-sourced at https://github.com/RedHatEMEA/satools. I hope they are useful to you. Questions, comments, feature requests, patches (hey, it is open source after all), pull requests and cake are all welcomed at jminter@redhat.com.

Warning

Please note that the SA tools presentation search tool is not yet production quality. Some features are not yet fully implemented and in particular the presentation creation code is incomplete and may return an error causing your presentation to be lost. You are warned!

Introduction

The SA tools presentation search portal consists of three windows: a filesystem viewer (on the left), the main search and search results window, and a presentation creator (at the bottom).

Filesystem viewer

The filesystem viewer allows quick access to all presentations contained within the synchronised repositories. Presentation text searches (see below) can be restricted to particular folders or folder trees within the filesystem by selecting the Search in "xyz" or Search in "xyz" and subfolders entries from the folder context menu (right-click the folder).

Once selected, a presentation file can be clicked in order to view its slides in the search results window. Alternatively, the original presentation file can be downloaded via the context menu (right-click the presentation and select Download presentation...).

Search and search results

The search box allows full text search of all of the indexed presentation slides. Up to 500 slides are returned at any time, with most recently saved presentations first. The query syntax and search modifiers are documented below.

When a search is complete, thumbnails of search results appear underneath the search box. Double-click a thumbnail to view a larger version of the slide in a popup window. Within the popup window, the left and right keys move between search results; press the escape key or click the popup window to close it.

To find a presentation containing a particular slide of interest, select Find presentation in filesystem from the slide context menu (right-click the thumbnail). To view a presentation containing a particular slide, select Show presentation, and to directly download an original presentation containing a particular slide, select Download presentation....

Presentation creation

Custom presentations can be made and downloaded by dragging slides from the search results and dropping them into the presentation creation window. Slides within the presentation creation window can be reordered by dragging and dropping them within the window, and can be removed by selecting Remove slide from the slide context menu (right-click).

Having completed a custom presentation, select Download presentation from the context menu (right-click) and the application will all being well generate a presentation odp file for download.

Query syntax and search modifiers

SA tools presentation search queries are formed of one or more of the following three basic query types:

  • General word or prefix queries. These return slides containing the given word, or in the case of a prefix query, any word which starts with the given prefix. Matching is case-insensitive only for non-extended latin characters (a-z), otherwise it is case sensitive, and the indexer does not strip accents.

    Return slides containing the word linux:

    linux

    Return slides containing a word which starts with the prefix lin. This will match all slides that contain the word linux, but also those that contain the words linear, linker, linguistic, etc.:

    lin*

  • Modifiers. The following modifiers further restrict search results: after:, before:, in:, is:, name:, path: and under:.

    Return slides where the presentation was modified after 1st February 2012 (note that the date format must be dd/mm/yyyy):

    after:01/02/2012

    Return slides where the presentation was modified before 1st February 2012 (note that the date format must be dd/mm/yyyy):

    before:01/02/2012

    Return slides contained within presentation files in /path/to/folder:

    in:/path/to/folder

    Return all slides from /path/to/presentation.odp:

    is:/path/to/presentation.odp

    Return all slides where the filename contains the text platform:

    name:platform

    Return all slides where the path contains the text platform:

    path:platform

    Return slides contained within presentation files in /path/to/folder or its subfolders:

    under:/path/to/folder

  • Quoted queries. These return slides containing words matching the given general word or prefix queries, in the specified order with no intervening words. Phrase queries are specified by enclosing one or more space-separated words or prefixes in double quotes. Modifiers may not be included in quoted queries.

    Return slides that contain the phrase linux applications:

    "linux applications"

    Return slides that contain a phrase matching lin* app*. As well as linux applications, this would match such common phrases as linoleum appliances or link apprentices, etc.:

    "lin* app*"

Basic queries of the above three types may be joined using the following four operators, in order of precedence (strongest first):

  • The NEAR operator. This returns slides that contain the two given positive general or positive quoted queries within a certain proximity of eachother. By default the NEAR operator matches when there are 10 or fewer intervening words; a different limit N may be specified by specifying an operator of the form NEAR(N). The order in which the given words or phrases appear in the slide does not have to be the same as the order in which they appear in the query.

    Return slides containing the words linux and kernel with not more than 10 intervening words:

    linux NEAR kernel

    Return slides containing the words linux and kernel with not more than 5 intervening words:

    linux NEAR(5) kernel

    Return slides contain the phrase operating system and word linux with not more than 2 words separating the two:

    linux NEAR(2) "operating system"

  • Parentheses. These group query sub-expressions to override operator precedence.

    Return slides that contain both the words linux and scalability, or which contain the word performance:

    linux scalability OR performance

    Return slides that contain the word linux, and which also contain the word scalability or which also contain the word performance:

    linux (scalability OR performance)

  • The NOT operator. This negates the sense of the query which follows.

    Return slides that contain the word linux and not the word windows:

    linux NOT windows

  • The implicit and operator. This returns slides that match both of the given subqueries. As may by now be clear, the and operator is not actually written, but is implicitly understood when subqueries are concatenated.

    Return slides containing the word linux, modified after 1st February 2012:

    after:01/02/2012 linux

  • The OR operator. This returns slides that match at least one of the two given subqueries.

    Return slides that contain the word linux or the word windows, or both:

    linux OR windows
